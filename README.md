# Observer-Timekeeper_V1.2
1.1版本未进行用户相关数据持久化，为保障用户体验，1.2版本新开一个静态仓库存放


本次 1.2 版本在保留 v1.1 核心框架的前提下，新增以下四项正式需求：

1. **设定开始时间选项 / 单次观测时长选项**  
   用户可以自行设定一次观测周期的现实时间，从 **15 分钟到 6 小时** 之间可选。  
   当用户点击“开始观测”后，本次观测会以该时长为结算目标：
   - 若用户下次进入网站时，距离开始时间尚未达到该设定时长，则**不能获得新的观测结果**；
   - 若超出用户选定的时长，则**仍按选定时长结算**；
   - 用户可随时点击暂停，中断本轮观测。

2. **加入“已开始观测年份”显示**  
   在星球图或世界图案上方显示该观测对象“开始被观测的真实年份 / 日期”，以增强用户的现实感。  
   同时保留下方“真实世界累计观测时长”，形成“现实观测历史”与“星球内部演化时间”并置的感觉。

3. **加入成就系统**  
   每个时期根据：
   - 特殊抉择；
   - 稀有普通观测事件；
   - 阶段完成结果；
   - 星球类型；
   - 面板参数；
   来解锁不同成就。  
   每个成就可点击查看，并展示：
   - 成就名称；
   - 插画；
   - 成就说明；
   - 解锁条件；
   - 解锁时间。  
   每个阶段结束后，还要设计若干“分歧结局成就”。

4. **加入更完整的插图系统**  
   不同类型的星球在不同阶段，都应拥有不同的大图，且要明显传达：
   - 星球环境
   - 当前演化阶段
   - 是否已有生命
   - 是否已有文明
   - 当前文明风格  
   可采用：
   - AI 生图；
   - 公开素材库；
   - 程序化 SVG 作为兜底。  
   本版将正式把插图系统列为产品层级模块，而不再只是可选美化元素。

# 二、网站核心定位（v1.2 版）

v1.2 版本的网站定位依然是：

**“以现实中的学习/工作时间为驱动，作为高维观测员长期追踪一颗星球，从形成、生命、文明到星际前夜的演化过程。”**

但与 v1.1 相比，本版更强调以下三种体验：

## 2.1 观测节奏感

用户不再只是“有空就点开始，回来就看变化”，而是：

- 可以主动设定本次观测时长；
- 知道自己是在做一次 **15 分钟短观测**，还是一次 **6 小时长观测**；
- 未到时间前返回，不会强行给结果，从而形成“等待观测窗口完成”的期待感。

## 2.2 现实锚点感

通过显示：
- 已开始观测年份 / 日期；
- 累计真实观测时长；
- 当前样本观测轮次；
让用户对“这颗星球被我观测了多久”有现实感，而不只是看着几千万年、几亿年抽象流逝。

## 2.3 收藏与收集感

引入成就系统后，网站不再只是“看简报”，而是多了一层长期收集动机：

- 解锁稀有事件成就；
- 收集阶段分支成就；
- 收集不同类型星球的专属成就；
- 查看对应插画与档案描述。
